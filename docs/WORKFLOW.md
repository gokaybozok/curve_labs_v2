# Workflow Guide

This document describes the development workflow for the Lab Derivative project.

## Git Workflow

1. Create a feature branch from `main`
2. Make changes and commit with descriptive messages
3. Open a Pull Request for review
4. Merge after CI passes and approval

## Development Cycle

1. Pick a task from the project board
2. Implement changes in a feature branch
3. Test locally before pushing
4. Create PR and request review

## Deployment

- **Staging**: Automatically deployed on PR merge to `develop` (if applicable)
- **Production**: Deployed on merge to `main` after CI passes
